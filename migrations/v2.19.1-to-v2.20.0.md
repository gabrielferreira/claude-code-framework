# Migration v2.19.1 → v2.20.0

> Data: 2026-04-08
> Tipo: minor

## Resumo

Adição de GitHub Actions CI com validação automática do framework e notificação no Google Chat. Nenhum arquivo de projeto foi alterado — mudanças são internas ao repositório do framework.

## Pre-requisitos

- Versao atual do projeto: v2.19.1
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 🔄 Overwrite (substituir arquivo inteiro)

#### 1. `.claude-plugin/plugin.json`

**O que mudou:** Campo `version` atualizado para 2.20.0.

**Como aplicar:**
```bash
cp {framework_path}/.claude-plugin/plugin.json {project_path}/.claude-plugin/plugin.json
```

**Impacto se nao aplicar:** Versao reportada pelo plugin fica desatualizada.

---

### 🆕 Arquivos novos (internos ao framework — não copiados para projetos)

#### 2. `.github/workflows/ci.yml` (novo)

**O que e:** Workflow de validação automática. Roda em todo push e PR.

**Checks:** framework-tags, version sync, source-template sync, migration exists (em tag push).

**Relevante se:** Você fez fork do framework e quer CI automático no seu repo.

**Como instalar:** Copiar do framework (só aplicável a repos do próprio framework).

#### 3. `scripts/check-sync.sh` (novo)

**O que e:** Script que verifica se todos os arquivos em `skills/setup-framework/templates/` estão em sincronia com seus sources, usando o `framework-file:` tag como mapa.

**Como usar:**
```bash
bash scripts/check-sync.sh
```

**Relevante se:** Você mantém um fork do framework e quer rodar a validação localmente antes de releases.
