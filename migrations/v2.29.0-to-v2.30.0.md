# Migration v2.29.0 → v2.30.0

> **Como aplicar:** siga cada seção em ordem. Arquivos `overwrite` podem ser copiados direto. Seções `structural` precisam de merge manual.

---

## Arquivos removidos

Estes arquivos foram removidos da distribuição e podem ser deletados do projeto:

### Se a skill foi instalada por projeto (`cp -r .../setup-framework .claude/skills/setup-framework`)

```bash
# Templates redundantes — não são mais distribuídos
rm -f .claude/skills/setup-framework/templates/CLAUDE.template.md
rm -f .claude/skills/setup-framework/templates/SPECS_INDEX.template.md
rm -f .claude/skills/setup-framework/templates/migrations/MIGRATION_TEMPLATE.md

# Migrations históricas dos templates — não são mais distribuídas
rm -f .claude/skills/setup-framework/templates/migrations/v*.md
```

### Migrations históricas no projeto (se acumularam via updates anteriores)

O update-framework agora distribui apenas as migrations do gap atual. Migrations de versões já passadas podem ser deletadas:

```bash
# Manter apenas: migrations/README.md e migrations do seu gap atual
# Deletar: qualquer v{X}-to-v{Y}.md onde Y <= sua versão atual instalada antes deste update
ls migrations/v*.md   # listar para decidir
```

## Skills (structural)

### `skills/setup-framework/SKILL.md` (se instalada por projeto)

Mudanças de comportamento — aplicar manualmente se a skill está em `.claude/skills/setup-framework/SKILL.md`:

**Fase 0 — Localizar templates (sentinela):**
- Antes: verificava `${CLAUDE_SKILL_DIR}/templates/CLAUDE.template.md`
- Depois: verifica `${CLAUDE_SKILL_DIR}/templates/CLAUDE.md`

Substituir na seção "Fase 0":
```
# Antes
verificar se existem templates embutidos em ${CLAUDE_SKILL_DIR}/templates/CLAUDE.template.md
Validar que o path informado contem CLAUDE.template.md na raiz

# Depois
verificar se existem templates embutidos em ${CLAUDE_SKILL_DIR}/templates/CLAUDE.md
Validar que o path informado contem CLAUDE.md na raiz
```

**Seção 3.2 CLAUDE.md:**
- Antes: `Usar ${FRAMEWORK_PATH}/CLAUDE.template.md como base`
- Depois: `Usar ${FRAMEWORK_PATH}/CLAUDE.md como base`

**Seção 3.4 SPECS_INDEX.md:**
- Antes: `Usar ${FRAMEWORK_PATH}/SPECS_INDEX.template.md como base`
- Depois: `Usar ${FRAMEWORK_PATH}/SPECS_INDEX.md como base`

**Nova seção 3.13 Migrations:**
Copiar apenas `migrations/README.md`. Não copiar `v{X}-to-v{Y}.md` — projeto novo não tem histórico a migrar.

### `skills/update-framework/SKILL.md`

Mudança na seção 3.1 Aplicar overwrite — tratamento especial de migrations:

- **Antes:** copiava todos os arquivos overwrite (incluindo todas as migrations)
- **Depois:** para `migrations/v{X}-to-v{Y}.md`, copia apenas migrations do gap atual (versão instalada → nova versão) e deleta migrations antigas do projeto

Atualizar a skill se customizada localmente.
