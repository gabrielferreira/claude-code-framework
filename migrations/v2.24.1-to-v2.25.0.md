# Migration v2.24.1 → v2.25.0

> Data: 2026-04-09
> Tipo: minor

## Resumo

Nova skill `research` com protocolo estruturado de investigação antes do planning (CE4). Terminologia de execução unificada: "Fase N" → "Wave N" na ordem de execução (CE2). Correção ortográfica "Parallelizável" → "Paralelizável".

## Pre-requisitos

- Versao atual do projeto: v2.24.1
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 🆕 Arquivos novos

#### 1. `.claude/skills/research/README.md` (novo)

**O que e:** Skill com protocolo estruturado de investigação antes do planning. Define 6 eixos de pesquisa (stack, código existente, patterns de reuso, dependências, riscos, gaps), formato de saída para `{id}-research.md` e regras de integração com execution-plan.

**Relevante se:** Projeto trabalha com itens Grande/Complexo, brownfield com patterns desconhecidos, ou domínios novos.

**Como instalar:** Copiar `skills/research/README.md` para `.claude/skills/research/README.md`.

```bash
mkdir -p .claude/skills/research
cp {framework_path}/skills/research/README.md .claude/skills/research/README.md
```

**Se decidir nao instalar:** Fase "research" no spec-driven continua funcionando, mas sem protocolo estruturado — o agente vai explorar de forma ad-hoc sem checklist ou formato de saída padronizado.

### 🔧 Content patches — mudancas dentro de secoes existentes

#### 2. `.claude/skills/spec-driven/README.md` — seção "Detalhamento das fases"

**Motivo:** Fase Research expandida para referenciar a nova skill research e descrever que achados alimentam a fase Plan.

**Research (sessão 1 — exploratória, Grande/Complexo):**

- Salvar achados em `.claude/specs/{id}-research.md` (descartável, só referência)

**Texto novo:**
```markdown
**Research (sessão 1 — exploratória, Grande/Complexo):**
- Seguir protocolo da skill `.claude/skills/research/README.md`
- Investigar 6 eixos: stack, código existente, patterns de reuso, dependências, riscos, gaps
- Salvar achados em `.claude/specs/{id}-research.md` (descartável, só referência)
- Output alimenta a fase Plan: spec, design doc e execution-plan são escritos a partir dos achados
- Esta sessão vai consumir muitos tokens explorando — é esperado
```

**Impacto se nao aplicar:** Research continua genérico ("explorar codebase") em vez de seguir protocolo estruturado.

#### 6. `.claude/skills/execution-plan/README.md` — seção "Ordem de execução" renomeada para "Waves de execução"

**Motivo:** Terminologia unificada "Fase" → "Wave" para disambiguar de fases do lifecycle (research/plan/execute/verify). Adicionado algoritmo de derivação de waves.

**Texto antigo:**
```markdown
### 4. Ordem de execução

Fase 1 (sequencial): Parte 1 — backend auth
Fase 2 (paralelo): Parte 2 + Parte 3 — frontend + testes E2E (sem overlap)
Fase 3 (sequencial): Integração — verificar que partes se encaixam

Justificar paralelismo: "Parte 2 e 3 não compartilham arquivos" ou "Parte 2 depende de Parte 1".
```

**Texto novo:**
```markdown
### 4. Waves de execução

Derivar waves a partir do grafo de dependências da spec (ou da decomposição em partes):

- **Wave 1:** partes sem dependências
- **Wave 2:** partes cujas dependências estão todas na Wave 1
- **Wave N:** partes cujas dependências estão em waves anteriores
- Dentro de cada wave: partes sem overlap de arquivos podem rodar em **paralelo** (`[P]`)

Wave 1 (sequencial): Parte 1 — backend auth
Wave 2 (paralelo):   Parte 2 [P] + Parte 3 [P] — frontend + testes E2E (sem overlap)
Wave 3 (integração):  Verificar que partes se encaixam

Justificar paralelismo: "Parte 2 e 3 não compartilham arquivos" ou "Parte 2 depende de Parte 1".

> Se o projeto usa sub-agents, as waves do plan mapeiam diretamente para as waves de despacho da skill context-fresh.
```

**Impacto se nao aplicar:** Execution-plan usa terminologia "Fase" ambígua e não tem algoritmo de derivação de waves.

#### 7. Correção ortográfica: "Parallelizável" → "Paralelizável"

**Motivo:** Português correto usa 1 L. Afeta: `specs/TEMPLATE.md`, `skills/context-fresh/README.md`, `skills/execution-plan/README.md`, `skills/spec-creator/SKILL.md`, `docs/SPEC_EXAMPLE.md`.

**Como aplicar:**
```bash
grep -rl "Parallelizável\|Parallelizavel" --include="*.md" .claude/ specs/ | xargs sed -i '' 's/Parallelizável/Paralelizável/g; s/Parallelizavel/Paralelizavel/g'
```

**Impacto se nao aplicar:** Erro ortográfico cosmético. Não afeta funcionalidade.
