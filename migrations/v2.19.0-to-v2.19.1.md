# Migration v2.19.0 → v2.19.1

> Data: 2026-04-08
> Tipo: patch

## Resumo

Correções no fluxo Notion e reforço de regras de processo: Responsavel preenchido ao concluir spec, validação de properties na criação, delegação obrigatória para sub-agents após plano pronto, e PR obrigatório nos checklists de entrega.

## Pre-requisitos

- Versao atual do projeto: v2.19.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 🔄 Overwrite (substituir arquivo inteiro)

#### 1. `.claude-plugin/plugin.json`

**O que mudou:** Campo `version` atualizado para 2.19.1.

**Como aplicar:**
```bash
cp {framework_path}/.claude-plugin/plugin.json {project_path}/.claude-plugin/plugin.json
```

**Impacto se nao aplicar:** Versao reportada pelo plugin fica desatualizada.

---

### 📝 Structural (adicionar/remover secoes)

#### 2. `.claude/skills/backlog-update/SKILL.md`

**O que mudou:** Ação `done` em modo Notion agora resolve e preenche `Responsavel`.

Substituir o bloco da ação `done` (Modo Notion):

```markdown
#### Ação: `done`

1. **Buscar a spec no Notion** — usar `notion-search` ou buscar por título/ID na database
2. **Resolver identidade do responsável** — chamar `notion-get-users` com `user_id: "self"` para obter o usuário da sessão atual
3. **Atualizar propriedades** via `notion-update-page`:
   \```
   command: "update_properties"
   properties: {
     "Status": "concluída",
     "date:Concluída em:start": "{data de hoje YYYY-MM-DD}",
     "date:Concluída em:is_datetime": 0,
     "Responsavel": "{user_id obtido no passo 2}"
   }
   \```
4. Se existir SPECS_INDEX.md local, atualizar status lá também
```

E na seção Regras, substituir:
`- **Modo Notion:** atualizar Status e Concluída em direto na página do Notion`
por:
`- **Modo Notion:** ao concluir (done), atualizar Status, Concluída em **e Responsavel** (via notion-get-users self) direto na página do Notion`

**Impacto se nao aplicar:** Responsavel fica vazio ao concluir specs no Notion.

---

#### 3. `.claude/skills/spec-creator/SKILL.md`

**O que mudou:** Step 7 do modo Notion agora valida properties além do body.

Substituir o step 7 do Modo Notion para incluir validação de properties antes da validação de conteúdo (ver arquivo fonte para conteúdo completo: `skills/spec-creator/SKILL.md`, step 7).

**Impacto se nao aplicar:** Campos obrigatórios (Tipo, Severidade, Fase, etc.) podem ficar vazios sem aviso.

---

#### 4. `.claude/skills/execution-plan/README.md`

**Regra adicionada (nova regra 3):**

```markdown
3. **Plano pronto = delegar implementação.** Após o execution-plan estar concluído na sessão principal, NÃO implementar no mesmo contexto — delegar cada parte para sub-agents. A sessão principal planeja, orquestra e integra; sub-agents executam. Implementar no contexto de planejamento mistura dois papéis e degrada o contexto.
```

(As regras 4-6 anteriores passam a ser 5-7.)

**Impacto se nao aplicar:** Sessão principal continua implementando no mesmo contexto onde planejou, degradando a qualidade da implementação.

---

#### 5. `.claude/skills/spec-driven/README.md`

**Mudanças:**

**5a. Tabela de fluxo** — adicionar sub-agents para Médio e Grande, e nota de delegação:

```markdown
| **Médio** | ... | Backlog → spec → execution-plan → **sub-agents** → integra → commit |
| **Grande** | ... | Backlog → spec → design → execution-plan → **sub-agents** → integra → commit |

> **Regra de delegação (Médio+):** após o execution-plan estar pronto na sessão principal, **não implementar no mesmo contexto** — delegar cada parte para sub-agents.
```

**5b. Fluxo RPI** — renomear de "Grande/Complexo" para "Médio+" e atualizar Implement:

```markdown
## Fluxo RPI — Research, Plan, Implement (Médio+)
...
**Implement:**
- Delegar cada parte do execution-plan para **sub-agents** — não implementar no contexto principal
...
**Princípio:** contexto de implementação recebe APENAS o necessário para executar. Quem planejou não implementa — delega.
```

**5c. Pós-implementação** — adicionar passo 4:

```markdown
4. **Abrir PR** — nunca push direto para `main`. Título segue Conventional Commits; descrição inclui link/referência para a spec implementada.
```

**Impacto se nao aplicar:** Sessão planeja e implementa no mesmo contexto; PR pode ser esquecido.

---

#### 6. `.claude/skills/definition-of-done/README.md`

**Seção adicionada** após `### Commits`:

```markdown
### Entrega
- [ ] **Abrir PR** para `main` (ou branch de integração) — nunca push direto
- [ ] Título do PR segue Conventional Commits
- [ ] Descrição do PR inclui contexto da mudança e link/referência para a spec
```

**Impacto se nao aplicar:** PR pode ser esquecido no checklist de entrega.
