# Migration v2.9.0 → v2.10.0

> Data: 2026-04-02
> Tipo: minor

## Resumo

Setup e update agora detectam o perfil do projeto (stack, DB, frontend, PRD, Notion) e filtram agents e skills por relevancia, em vez de copiar tudo cegamente. Agents condicionais (`seo-audit`, `component-audit`, `product-review`) e skills condicionais (`dba-review`, `ux-review`, `seo-performance`) so sao instalados se o perfil bate ou o usuario aceitar explicitamente. O `/update-framework` executa essa deteccao na nova Fase 0.4, antes de propor qualquer mudanca.

## Pre-requisitos

- Versao atual do projeto: v2.9.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

### 📝 Structural (adicionar/remover secoes)

> Estes arquivos tem conteudo customizado pelo projeto. Apenas secoes novas/removidas sao listadas.

#### 1. `.claude/skills/setup-framework/SKILL.md`

**Secoes adicionadas:**

Adicionar apos a Fase 0 (deteccao inicial), ou criar como nova fase antes da Fase 1:

```markdown
## Fase 0.4. Deteccao de perfil do projeto

Antes de propor instalacao de agents e skills, detectar o perfil do projeto:

- **Stack** — linguagem principal, framework backend
- **Banco de dados** — tipo (SQL/NoSQL/nenhum)
- **Frontend** — presente? qual framework?
- **PRDs** — diretorio `prds/` existe?
- **Notion** — integracao configurada no CLAUDE.md?

Com base no perfil detectado, filtrar agents e skills condicionais:

- `seo-audit` — instalar apenas se frontend presente
- `component-audit` — instalar apenas se frontend presente
- `product-review` — instalar apenas se PRD presente ou Notion configurado
- `dba-review` — instalar apenas se banco de dados presente
- `ux-review` — instalar apenas se frontend presente
- `seo-performance` — instalar apenas se frontend presente

Para cada item condicional que nao bate o perfil, perguntar ao usuario se deseja instalar mesmo assim antes de pular.
```

**Secoes modificadas:**

Na Fase 1 (ou equivalente de instalacao de agents/skills), substituir a logica de "copiar tudo" por:

```markdown
Instalar agents e skills conforme o perfil detectado na Fase 0.4.
Agents e skills condicionais: instalar apenas se perfil bate ou usuario aceitou.
```

**Impacto se nao aplicar:** O `/setup-framework` continua instalando todos os agents e skills sem filtro de perfil — comportamento anterior (v2.9.0), funcional mas menos preciso para o contexto do projeto.

## Fase 0.4. Deteccao de contexto do projeto

Antes de propor mudancas, detectar o contexto atual do projeto:

Usar esse contexto para filtrar quais agents e skills condicionais sao relevantes ao propor updates. Nao propor instalacao de `seo-audit`, `component-audit` ou `product-review` se o perfil nao bate, a menos que o usuario solicite explicitamente.

**Impacto se nao aplicar:** O `/update-framework` propoe updates para todos os agents e skills, incluindo os que nao fazem sentido para o perfil do projeto — comportamento anterior (v2.9.0), funcional mas com ruido desnecessario.

### 👀 Manual (decisao humana)

> Estes arquivos sao altamente customizados. Revise o diff e decida o que aplicar.

#### 1. `CLAUDE.md` do projeto (gerado a partir de `CLAUDE.template.md`)

**O que mudou no template:**

```diff
+## Perfil do projeto
+
+Informacoes usadas pelo `/setup-framework` e `/update-framework` para filtrar
+agents e skills condicionais:
+
+- **Stack:** {ex: Node.js + Express, Python + FastAPI}
+- **Banco de dados:** {ex: PostgreSQL, MongoDB, nenhum}
+- **Frontend:** {ex: React, Vue, nenhum}
+- **PRDs:** {sim | nao}
+- **Notion:** {sim | nao}
```

**Sugestao:** Adicionar a secao `## Perfil do projeto` no `CLAUDE.md` do projeto com os dados reais do projeto. Isso permite que o setup e o update detectem o perfil automaticamente sem precisar inferir pelo codigo. Se omitir, o framework tenta inferir pelo contexto, mas a secao explicita e mais confiavel.
