# Migration v2.37.0 → v2.37.1

> Data: 2026-04-10
> Tipo: patch

## Resumo

Adicionado passo explicito de substituicao de `{NOME_DO_PROJETO}` no setup e update. Projetos ficavam com placeholder literal nos titulos de skills, docs e specs.

## Pre-requisitos

- Versao atual do projeto: v2.37.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

Esta versao corrige um bug nas instrucoes do setup/update-framework. **Nenhuma acao necessaria em projetos ja instalados** — os placeholders `{NOME_DO_PROJETO}` que ja estavam nos arquivos continuam la e podem ser substituidos manualmente:

```bash
# Substituir {NOME_DO_PROJETO} pelo nome real em todos os arquivos do projeto
grep -rl "{NOME_DO_PROJETO}" --include="*.md" . | xargs sed -i 's/{NOME_DO_PROJETO}/NOME REAL DO PROJETO/g'
```

A correcao garante que futuros `/setup-framework` e `/update-framework` facam essa substituicao automaticamente.

---

## Framework-tags

Apos aplicar as mudancas, atualize os framework-tags nos arquivos tocados:

```bash
grep -rl "framework-tag: v2.37.0" --include="*.md" . | xargs sed -i 's/framework-tag: v2.37.0/framework-tag: v2.37.1/g'
```
