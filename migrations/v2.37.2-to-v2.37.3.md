# Migration v2.37.2 → v2.37.3

> Data: 2026-04-10
> Tipo: patch

## Resumo

Reposicionamento do framework de "spec-driven" / "framework de documentacao" para **harness engineering framework**. Spec-driven continua como skill — reposicionado de "o que o framework e" para "entry point do fluxo de conhecimento".

## Pre-requisitos

- Versao atual do projeto: v2.37.2
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 🔧 Content patches — mudancas dentro de secoes existentes

#### 1. `docs/QUICK_START.md` — intro

**Motivo:** intro mencionava "fluxo spec-driven" como proposito principal.

**Texto antigo:**
```markdown
> Em 5 minutos voce tera: CLAUDE.md configurado, backlog pronto, skills de dominio instaladas e um fluxo spec-driven funcionando no seu projeto.
```

**Texto novo:**
```markdown
> Em 5 minutos voce tera: CLAUDE.md configurado, skills de dominio instaladas, agents de auditoria prontos e o harness completo funcionando no seu projeto.
```

#### 2. `docs/SKILLS_MAP.md` — titulo da secao de fluxo

**Motivo:** "Fluxo principal" nao refletia o conceito de orquestracao.

**Texto antigo:**
```markdown
## Fluxo principal (ordem recomendada)
```

**Texto novo:**
```markdown
## Pipeline de orquestracao (ordem recomendada)
```

#### 3. `docs/SKILLS_GUIDE.md` — intro

**Motivo:** intro nao contextualizava skills como camada de expertise do harness.

**Texto antigo:**
```markdown
> Referência descritiva de todas as skills do framework: o que cada uma faz, quando usar e o que produz.
> Use para decidir qual skill invocar antes de iniciar uma tarefa.
>
> Para dependências e ordem de execução, consultar `docs/SKILLS_MAP.md`.
```

**Texto novo:**
```markdown
> Skills formam a camada de *expertise* do harness — checklists por dominio que o Claude Code consulta antes de agir.
> Referencia descritiva de todas as skills: o que cada uma faz, quando usar e o que produz.
>
> Para dependencias e ordem de execucao, consultar `docs/SKILLS_MAP.md`.
```

#### 4. `docs/SPEC_DRIVEN_GUIDE.md` — preamble adicionado

**Motivo:** guia parecia definir o framework inteiro quando e apenas a camada de conhecimento.

Adicionar logo apos o titulo `# Spec-Driven Development com AI: a pratica do SPECS_INDEX`, antes de `## O problema`:

```markdown
> **Contexto no harness:** spec-driven e a camada de *conhecimento* do harness engineering framework. Define *o que* o Claude Code vai implementar e *por que*. As outras camadas — skills (expertise), agents (automacao), orquestracao, verificacao e continuidade — governam *como* ele opera. Este guia foca na camada de conhecimento.
```

#### 5. `docs/CONCEPTUAL_MAP.md` — descricao

**Texto antigo:**
```markdown
> Guia rapido para entender como os artefatos do framework se relacionam com a terminologia padrao da industria (Epics, Stories, Tasks).
```

**Texto novo:**
```markdown
> Guia rapido para entender como os artefatos do harness se relacionam com a terminologia padrao da industria (Epics, Stories, Tasks). Estes artefatos formam a camada de conhecimento do framework.
```
