# Migration v2.15.1 → v2.16.0

> Data: 2026-04-07
> Tipo: minor

## Resumo

Verificacao automatica pos-setup e pos-update para prevenir regressao de conteudo customizado. Skills de gestao (setup-framework, update-framework) nao sao mais copiadas para o projeto.

## Pre-requisitos

- Versao atual do projeto: v2.15.1
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 👀 Manual (decisao humana)

#### 1. Remover `setup-framework` e `update-framework` do projeto (se existirem)

Se `.claude/skills/setup-framework/` ou `.claude/skills/update-framework/` existem no projeto, remover — sao skills de gestao que devem ficar em `~/.claude/skills/` (pessoal) ou via plugin.

```bash
rm -rf .claude/skills/setup-framework/
rm -rf .claude/skills/update-framework/
```

**Impacto se nao remover:** Ocupam espaco desnecessario e podem confundir (o setup-framework inclui todos os templates do framework).

### 📝 Structural (adicionar/remover secoes)

#### 2. `.claude/skills/setup-framework/SKILL.md` (se instalado em `~/.claude/skills/`)

**O que mudou:**
- Fase 3.6: nota explicita de que setup-framework e update-framework NAO devem ser copiados para o projeto
- Fase 5a (nova): verificacao pos-geracao que valida customizacao de CODE_PATTERNS nas skills instaladas

**Como aplicar:** Atualizar via `install-skills.sh` ou copiar manualmente do framework atualizado.

#### 3. `.claude/skills/update-framework/SKILL.md` (se instalado em `~/.claude/skills/`)

**O que mudou:**
- Fase 3: regra absoluta de que structural NUNCA e substituido por cp
- Fase 3.2: backup obrigatorio ANTES de cada merge
- Fase 3.5 (nova): verificacao pos-aplicacao com restauracao automatica do backup se detectar regressao
- Algoritmo de merge structural reforcado com regra critica e criterios de deteccao de customizacao

**Como aplicar:** Atualizar via `install-skills.sh` ou copiar manualmente do framework atualizado.
