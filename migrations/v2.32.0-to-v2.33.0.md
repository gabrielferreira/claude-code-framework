# Migration v2.32.0 → v2.33.0

> Data: 2026-04-10
> Tipo: minor

## Resumo

Novo agent `debugger` para diagnóstico estruturado de falhas com hipóteses ranqueadas. Skills e agents existentes ganham exemplos concretos para C#, Dart e Rust. Listas de instalação/auditoria de agents corrigidas para incluir `plan-checker` (ausente desde SA2) e `debugger`.

## Pre-requisitos

- Versao atual do projeto: v2.32.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 🆕 Arquivos novos

#### 1. `.claude/agents/debugger.md` (novo)

**O que e:** Agent read-only que coleta contexto de falha automaticamente (erro, histórico git, STATE.md, ambiente) e produz diagnóstico estruturado com hipóteses ranqueadas, confiança global e próximos passos acionáveis.

**Relevante se:** projeto usa agents para auditoria e diagnóstico (sempre recomendado).

**Como instalar:** Copiar `agents/debugger.md` para `.claude/agents/debugger.md`.

```bash
cp {framework_path}/agents/debugger.md {project_path}/.claude/agents/debugger.md
```

**Se decidir nao instalar:** diagnóstico de falhas fica manual — sem impacto em outros fluxos.

### 🔧 Content patches — mudancas dentro de secoes existentes

> O merge structural preserva conteudo customizado — NAO atualiza conteudo dentro de secoes existentes.
> Estas mudancas precisam ser aplicadas manualmente (ou aceitas via prompt do `/update-framework`).

#### 2. `.claude/agents/coverage-check.md` — seção `## Input`

**Motivo:** comandos de coverage expandidos para C#, Dart e Rust.

**Texto antigo** (encontrar e substituir):
```markdown
- Resultado de coverage (`npm run test:cov`, `npx vitest run --coverage`, etc.)
```

**Texto novo:**
```markdown
- Resultado de coverage — comando por linguagem:
  - JS/TS → `npm run test:cov`, `npx vitest run --coverage`
  - Python → `pytest --cov={src}/ --cov-report=term-missing`
  - Go → `go test ./... -coverprofile=coverage.out && go tool cover -html=coverage.out`
  - C# → `dotnet test --collect:"XPlat Code Coverage"` (requer coverlet)
  - Dart → `dart test --coverage=coverage && dart pub global run coverage:format_coverage --lcov -i coverage`
  - Rust → `cargo llvm-cov` ou `cargo tarpaulin --out Html`
```

**Impacto se nao aplicar:** agent funciona mas não lista comandos de coverage para stacks C#/Dart/Rust.

#### 3. `.claude/agents/coverage-check.md` — seção `## O que verificar` (item 2 e 5)

**Motivo:** extensões de arquivo e pragmas de exclusão expandidos para novas stacks.

**Texto antigo (item 2):**
```markdown
Identificar arquivos de código (.js, .jsx, .ts, .tsx, .py, .go) que:
```

**Texto novo (item 2):**
```markdown
Identificar arquivos de código (.js, .jsx, .ts, .tsx, .py, .go, .cs, .dart, .rs) que:
```

**Texto antigo (item 5):**
```markdown
Verificar `/* c8 ignore */`, `/* istanbul ignore */`, `// coverage:ignore`:
```

**Texto novo (item 5):**
```markdown
Verificar exclusões de coverage por linguagem — `/* c8 ignore */` / `/* istanbul ignore */` (JS/TS), `# pragma: no cover` (Python), `//nolint:...` (Go), `#[cfg(not(test))]` (Rust), `// coverage:ignore` (Dart), `[ExcludeFromCodeCoverage]` (C#):
```

**Impacto se nao aplicar:** agent não detecta arquivos sem teste e exclusões injustificadas em C#/Dart/Rust.

#### 4. `.claude/agents/test-generator.md` — seção `## Input` e `## Regras`

**Motivo:** stacks de teste e detecção automática expandidos para C#, Dart e Rust.

**Texto antigo (Input):**
```markdown
- Stack de testes do projeto (detectar automaticamente ou ler CLAUDE.md)
```

**Texto novo (Input):**
```markdown
- Stack de testes do projeto (detectar automaticamente ou ler CLAUDE.md):
  - JS/TS → Jest, Vitest, Mocha; mock: jest.mock / sinon
  - Python → pytest, unittest; mock: unittest.mock, pytest-mock
  - Go → testing, httptest; mock: interfaces + structs manuais
  - C# → xUnit, NUnit, MSTest; mock: Moq, NSubstitute
  - Dart → test package, flutter_test; mock: Mockito (Dart), @GenerateMocks
  - Rust → cargo test, tokio::test; mock: mockall (#[automock])
```

**Texto antigo (Regras):**
```markdown
- Detectar framework de teste do projeto (jest, vitest, mocha, pytest, etc.) automaticamente
```

**Texto novo (Regras):**
```markdown
- Detectar framework de teste do projeto automaticamente: jest/vitest/mocha (JS/TS), pytest (Python), xUnit/NUnit (C#), cargo test (Rust), dart:test (Dart), testing/httptest (Go)
```

**Impacto se nao aplicar:** agent gera testes mas sem referência de frameworks para C#/Dart/Rust.
