# Migration v2.5.0 → v2.6.0

> Data: 2026-04-02
> Tipo: minor

## Resumo

Release grande: sistema de PRD (opt-in), 3 skills novas, 3 agents novos (incluindo 2 action agents com `worktree: true`), 7 docs novos, padronizacao de severidade e placeholders em todos os agents e skills, e diversas melhorias de processo (validate-tags, dry-run no setup, checklist pos-release).

## Pre-requisitos

- Versao atual do projeto: v2.5.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

### 📝 Structural (adicionar/remover secoes)

> Estes arquivos tem conteudo customizado pelo projeto. Apenas secoes novas/removidas sao listadas — nao substituir o arquivo inteiro.

#### 1. Todas as skills existentes (`.claude/skills/*/README.md`)

**O que mudou:** Placeholders padronizados para o formato `{Adaptar: descricao}`; exemplos concretos adicionados nas skills `dba-review`, `mock-mode`, `code-quality`, `docs-sync`.

**Como aplicar:** Rodar `/update-framework` — o merge structural aplica automaticamente. Se aplicando manualmente, revisar cada skill e atualizar placeholders no formato antigo para `{Adaptar: descricao}`.

**Impacto se nao aplicar:** Skills funcionam, mas placeholders ficam com formato inconsistente.

---

#### 2. Todos os docs existentes (`docs/*.md`)

**O que mudou:** Conteudo incremental adicionado conforme melhorias de documentacao desta versao.

**Como aplicar:** Rodar `/update-framework` — merge structural aplicado automaticamente.

### 👀 Manual (decisao humana)

> Estes arquivos sao altamente customizados. Revise o diff e decida o que aplicar.

#### 1. `CLAUDE.md` do projeto (baseado em `CLAUDE.template.md`)

**O que mudou no template:**

```diff
+ ## Ordem de precedencia entre skills
+ (nova secao documentando qual skill tem prioridade quando ha conflito)

+ ## Padroes para criar skills
+ (checklist para adicao de novas skills ao projeto)

+ ## Padroes para criar agents
+ (checklist para adicao de novos agents ao projeto)

+ # Sistema de PRD (opt-in)
+ (secao condicional — so relevante se projeto adota PRDs)
```

**Sugestao:**
- **Ordem de precedencia entre skills:** Recomendado aplicar — evita confusao quando multiplas skills se sobrepoem.
- **Padroes para criar skills/agents:** Aplicar se a equipe cria skills ou agents proprios no projeto; pode ignorar se so usa os do framework.
- **Sistema de PRD:** Aplicar apenas se o projeto adota PRDs como artefato de processo. Seccao e opt-in por design.
