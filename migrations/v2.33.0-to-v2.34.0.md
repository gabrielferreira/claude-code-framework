### 🔀 Renames (novidade v2.34.0)

> Esta versão introduz o conceito de renames no MANIFEST. O `/update-framework` v2.34.0+ aplica renames automaticamente.
> Se estiver atualizando manualmente (sem `/update-framework`), siga os passos abaixo.

#### 1. `.claude/skills/resume/README.md` → `.claude/skills/resume/SKILL.md`

**Motivo:** skill convertida para slash command (`/resume`). Precisa de frontmatter YAML com `user_invocable: true`.

**Como aplicar manualmente:**

1. Renomear o arquivo:
   ```bash
   mv .claude/skills/resume/README.md .claude/skills/resume/SKILL.md
   ```

2. Adicionar frontmatter YAML no **topo** do arquivo (antes do framework-tag):
   ```yaml
   ---
   name: resume
   description: Retomada estruturada apos crash, timeout ou context limit via STATE.md e execution-plan
   user_invocable: true
   ---
   ```

3. Atualizar o framework-tag:
   ```
   <!-- framework-tag: v2.43.0 framework-file: skills/resume/SKILL.md -->
   ```

4. Atualizar o título:
   ```markdown
   # /resume — Retomada estruturada
   ```

**Impacto se nao aplicar:** `/resume` não será reconhecido como slash command — a skill continuará funcionando como referência passiva, mas o usuário não poderá invocar digitando `/resume`.

## Framework-tags

Apos aplicar as mudancas, atualize os framework-tags nos arquivos tocados:

```bash

grep -rl "framework-tag: v2.43.0" --include="*.md" . | xargs sed -i 's/framework-tag: v2.43.0/framework-tag: v2.43.0/g'
```
