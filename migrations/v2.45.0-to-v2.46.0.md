# Migration v2.45.0 → v2.46.0

> Data: 2026-04-15
> Tipo: minor

## Resumo

Framework Light Edition (MO9): modo dual light/full no setup e update, nova skill /upgrade-framework, MANIFEST com coluna Tier.

## Pre-requisitos

- Versao atual do projeto: v2.45.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 📝 Structural — secoes novas

#### 1. `skills/setup-framework/SKILL.md`

**O que mudou:**
- Fase 0 passo 5: nova pergunta light/full com re-run detection
- Fase 0 passo 6: light pula monorepo detection, oferece trocar para full se detectado
- Fase 1: nota sobre analise igual em ambos os modos
- Fase 2: questionario simplificado para light (3-4 perguntas)
- Fase 3: filtro por tier (MANIFEST) antes de criar arquivos, resolucao templates-light/ first
- SETUP_REPORT: campo `> Modo:`
- 3.2 CLAUDE.md: `<!-- framework-mode: light -->` para light

**Como aplicar:** o `/update-framework` aplica secoes novas automaticamente. Se nao funcionar, copiar do framework source.

#### 2. `skills/update-framework/SKILL.md`

**O que mudou:**
- Fase 0.4: detecta modo (SETUP_REPORT → CLAUDE.md → heuristica)
- Filtragem por tier: light+full+nao existe → skip silencioso
- UPDATE_REPORT: secao "Ignorados (modo light)"

**Como aplicar:** mesma logica do setup.

---

### 🆕 Arquivos novos

#### 3. `skills/upgrade-framework/SKILL.md`

**O que e:** skill `/upgrade-framework` que converte projetos de modo light para full aditivamente. 5 fases: validacao, inventario, perguntas (PRD, monorepo, delta markers, sub-agents), instalacao, auditoria.

**Como instalar:** disponibilizado via plugin ou copiar manualmente. Nao e instalado automaticamente no projeto — e skill de gestao como setup e update.

#### 4. `skills/setup-framework/templates-light/` (~10 arquivos)

**O que e:** templates simplificados para modo light. CLAUDE.md (menos secoes, 11 skills, 5 agents), TEMPLATE.md (6 secoes), backlog (3 secoes, 5 colunas), STATE.md (minimo), spec-driven, spec-creator, backlog-update, definition-of-done, QUICK_START, docs/README.

**Como instalar:** parte do framework source, usado automaticamente pelo setup quando modo=light.

---

### 🔧 Interno (nao requer acao do usuario)

#### 5. `MANIFEST.md`

**O que mudou:** coluna Tier adicionada em todas as tabelas (core/full/conditional). Usada pelo setup para filtrar arquivos por modo e pelo update para deteccao de tier.

#### 6. `scripts/check-sync.sh`

**O que mudou:** secao D valida que framework-tags em templates-light/ batem com templates/.
