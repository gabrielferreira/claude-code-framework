# Migration v2.49.1 → v2.49.2

> Data: 2026-04-16
> Tipo: patch

## Resumo

Release 100% framework-internal — **nenhuma ação necessária em projetos downstream**. Inclui fix do `validate-structure.sh` no macOS (BSD grep com locale C falhava em "NÃO"), item-specs retroativas para DF13/TQ5, estreia de `scripts/release.sh`, e registro de anti-padrões de auditoria em `.claude/AUDIT_ANTIPATTERNS.md`.

## Pre-requisitos

- Versao atual do projeto: v2.49.1
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

Todos os arquivos tocados nesta versão são internos ao repo do framework e estão listados em `MANIFEST.md` na seção "Scripts do framework (não copiados)" ou vivem em `.claude/item-specs/`. Nenhum deles é distribuído para projetos via `setup-framework` ou `update-framework`.

- `.claude/AUDIT_ANTIPATTERNS.md` — doc interno de anti-padrões de auditoria
- `.claude/item-specs/done/{DF13,TQ5,TQ10,OP4}.md` — specs retroativas e novas
- `.claude/item-specs/INDEX.md` — índice atualizado
- `BACKLOG.md`, `CHANGELOG.md`, `CLAUDE.md` (do framework), `MANIFEST.md` — docs de desenvolvimento do framework
- `LICENSE` — já era interno, sem impacto no projeto
- `scripts/release.sh` (novo, framework-internal)
- `scripts/validate-structure.sh` (fix interno, framework-internal)

**Projetos que instalaram o framework não precisam fazer nada para esta versão.** Quando a próxima release com mudanças distribuídas sair, o gap `v2.49.2 → v{próxima}` cobrirá o que for relevante.

## Framework-tags

Para manter rastreabilidade mesmo sem ação funcional, atualize os framework-tags se quiser alinhar o projeto com a versão atual:

```bash
# Atualizar todos os framework-tags de v2.49.1 para v2.49.2
grep -rl "framework-tag: v2.49.1" --include="*.md" . | xargs sed -i 's/framework-tag: v2.49.1/framework-tag: v2.49.2/g'
```

Opcional — o `/update-framework` fará isso automaticamente na próxima atualização.
