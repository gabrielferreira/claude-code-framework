# Migration v2.20.0 → v2.21.0

> Data: 2026-04-08
> Tipo: minor

## Resumo

Unifica fluxos de complexidade, adiciona enforcement de plan para Médio+, e muda estratégia de update de agents/templates de overwrite para structural (preservando customizações do projeto). Introduz mecanismo de content patches para mudanças intra-seção.

## Pre-requisitos

- Versao atual do projeto: v2.20.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

#### 2. `.claude/skills/spec-creator/SKILL.md`

```markdown
9. **Próximos passos obrigatórios** (informar conforme complexidade):

   | Complexidade | Próximos passos |
   |---|---|
   | **Pequeno** | Aprovar spec → ler spec-driven → implementar → testar → commit |
```

### 🔧 Content patches — mudancas dentro de secoes existentes

> O merge structural preserva conteudo customizado — NAO atualiza conteudo dentro de secoes existentes.
> Estas mudancas precisam ser aplicadas manualmente (ou aceitas via prompt do `/update-framework`).

#### 3. `.claude/skills/spec-driven/README.md` — secao "Fluxo: da demanda ao código"

**Motivo:** Tabela de classificação reescrita — Pequeno agora cria spec light (antes só backlog). Fluxo Médio+ inclui execution-plan. Sub-agents condicional.

**Texto antigo** (encontrar e substituir):
```markdown
| **Pequeno** | ≤3 arquivos, <30min, sem regra de negócio | Só entrada no backlog | Backlog → implementa → testa → commit |
```

**Texto novo:**
```markdown
| **Pequeno** | ≤3 arquivos, <30min, sem regra de negócio | Spec light (contexto + critério mínimo) | Backlog → spec → implementa → testa → commit |
```

**Impacto se nao aplicar:** Framework continua dizendo que Pequeno não precisa de spec.

#### 4. `.claude/skills/spec-driven/README.md` — apos tabela de classificação

**Motivo:** Gate obrigatório para Médio+ e regra de delegação condicional.

**Texto antigo:**
```markdown
> **Regra de delegação (Médio+):** após o execution-plan estar pronto na sessão principal, **não implementar no mesmo contexto** — delegar cada parte para sub-agents.
```

**Texto novo:**
```markdown
> **Toda mudança tem spec.** A complexidade determina o nível de detalhe, não se a spec existe.

> **Regra de delegação (Médio+, se o projeto usa sub-agents):** após o execution-plan estar pronto na sessão principal, **não implementar no mesmo contexto** — delegar cada parte para sub-agents. **Se o projeto não usa sub-agents:** implementar sequencialmente seguindo a ordem do execution-plan.

> **Gate obrigatório (Médio+):** Antes de escrever a primeira linha de código, deve existir:
> 1. Spec com status `aprovada` (não `rascunho`)
> 2. Execution plan escrito (skill execution-plan) — plano mental não conta
> 3. Se o projeto usa sub-agents: decomposição em partes com briefing
>
> Se qualquer item estiver faltando → **PARAR e completar.**
```

**Impacto se nao aplicar:** Sem gate de enforcement — dev pode pular plan.

#### 5. `.claude/skills/spec-driven/README.md` — secao "TDD obrigatório"

**Motivo:** TDD tornada condicional + exceções unificadas.

**Texto antigo:**
```markdown
**Exceção para Pequeno:** mudanças classificadas como Pequeno (≤3 arquivos, <30min, sem regra de negócio) não precisam de spec formal, mas o teste de regressão é criado ANTES do fix e a entrada no backlog é obrigatória.
```

**Texto novo:**
```markdown
**Exceções:**
- **Pequeno (≤3 arquivos, <30min, sem regra de negócio):** teste de regressão ANTES do fix, mas spec light é suficiente (sem spec formal completa).
- **Bug urgente em produção (<30min):** implementar fix + criar teste de regressão imediatamente após. Documentar no commit por que o teste veio depois.
```

**Impacto se nao aplicar:** Exceções de TDD continuam inconsistentes entre arquivos.

#### 6. `.claude/skills/spec-driven/README.md` — secao "Complexidade" do backlog

**Motivo:** Emojis de complexidade diferenciados para não conflitar com severidade.

**Texto antigo:**
```markdown
| 🟢 | Baixa | 1-2 arquivos, mudança pontual, <30 min |
| 🟡 | Média | 3-5 arquivos, lógica nova moderada, 1-3h |
```

```markdown
| ⬛ | Complexo | Ambiguidade, domínio novo, >20 tasks |
```

**Impacto se nao aplicar:** Emojis de complexidade continuam conflitando com severidade (🟡 = Médio em ambos).

#### 7. `.claude/skills/execution-plan/README.md` — regra 3

**Motivo:** Sub-agents condicional — projetos sem sub-agents seguem modo sequencial.

**Texto antigo:**
```markdown
3. **Plano pronto = delegar implementação.** Após o execution-plan estar concluído na sessão principal, NÃO implementar no mesmo contexto — delegar cada parte para sub-agents.
```

**Texto novo:**


**Impacto se nao aplicar:** Execution-plan exige delegação a sub-agents mesmo em projetos que não usam.

## Nota sobre estratégias de update

A partir desta versão, **agents, templates de spec e backlog-format.md passam de `overwrite` para `structural`**. Isso significa que o `/update-framework` agora preserva customizações nesses arquivos (como `{Adaptar:}` preenchidos, `model:` editado, seções custom) em vez de sobrescrevê-los.

Se você tinha customizado algum agent e perdeu a customização em updates anteriores, este é o momento de reaplicar.
