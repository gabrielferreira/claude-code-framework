# Migration v2.4.0 → v2.5.0

> Data: 2026-04-01
> Tipo: minor

## Resumo

Adicionadas 4 skills novas (`security-review`, `seo-performance`, `syntax-check`, `golden-tests`) e 1 agent novo (`seo-audit`). Arquivos existentes de agents, docs e skills atualizados com framework-tag v2.5.0 e ajustes menores.

## Pre-requisitos

- Versao atual do projeto: v2.4.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 🆕 Arquivos novos

#### 1. `.claude/agents/seo-audit.md` (novo)

**O que e:** Agent autonomo que audita paginas publicas em SEO, Core Web Vitals (LCP, INP, CLS) e acessibilidade. Read-only — reporta, nao corrige. Deve ser complementado pela skill `seo-performance`.

**Relevante se:** projeto tem paginas publicas (landing page, blog, docs publicos).

**Como instalar:**

```bash
cp {framework_path}/agents/seo-audit.md {project_path}/.claude/agents/seo-audit.md
```

**Se decidir nao instalar:** auditoria de SEO fica sem agent dedicado. Ainda e possivel usar a skill `seo-performance` manualmente.

---

#### 2. `.claude/skills/security-review/README.md` (novo)

**O que e:** Checklist de seguranca para uso durante codificacao — antes de commitar codigo que toca rotas, middleware ou services. Cobre: novas rotas, endpoints de modificacao de dados, autenticacao, race conditions, integracoes externas, redirects e frontend.

**Diferenca do agent `security-audit`:** o agent faz auditoria read-only do sistema todo. Esta skill e guidance para quem esta escrevendo codigo.

**Relevante se:** projeto tem endpoints HTTP ou logica de autenticacao (praticamente todos).

**Como instalar:**

```bash
mkdir -p {project_path}/.claude/skills/security-review
cp {framework_path}/skills/security-review/README.md {project_path}/.claude/skills/security-review/README.md
```

Apos instalar, adaptar as secoes marcadas com `{Adaptar:}`:
- `## Regras absolutas do projeto` — listar as regras inviolaveis especificas do projeto
- `## Padroes de sanitizacao` — documentar as funcoes de sanitizacao da codebase
- `## Campos proibidos em respostas` — listar campos sensiveis que nunca devem sair na API

---

#### 3. `.claude/skills/seo-performance/README.md` (novo)

**O que e:** Checklists proativos de SEO, Core Web Vitals e acessibilidade para uso durante codificacao. Executar ao criar/modificar paginas publicas ou alterar bundle/build. Para auditoria completa, usar o agent `seo-audit`.

**Relevante se:** projeto tem paginas publicas com requisitos de SEO ou performance.

**Como instalar:**

```bash
mkdir -p {project_path}/.claude/skills/seo-performance
cp {framework_path}/skills/seo-performance/README.md {project_path}/.claude/skills/seo-performance/README.md
```

---

#### 4. `.claude/skills/syntax-check/README.md` (novo)

**O que e:** Checklist proativo de verificacao de sintaxe a rodar antes de cada commit. Inclui validacao de sintaxe por stack, padroes suspeitos (console.log esquecido, SQL injection via template literals, catch vazio, test.only), e consistencia de module system.

**Relevante se:** todos os projetos com codigo-fonte.

**Como instalar:**

```bash
mkdir -p {project_path}/.claude/skills/syntax-check
cp {framework_path}/skills/syntax-check/README.md {project_path}/.claude/skills/syntax-check/README.md
```

Adaptar os paths `{src}`, `{backend}`, `{frontend}`, `{tests}` e extensoes `{ext}` para a estrutura do projeto.

---

#### 5. `.claude/skills/golden-tests/README.md` (novo)

**O que e:** Skill para criar, modificar e revisar golden tests (snapshot tests) em endpoints HTTP e componentes UI. Inclui guia de custom serializers para normalizar dados dinamicos (timestamps, UUIDs, tokens) e regras para atualizacao segura de snapshots.

**Relevante se:** projeto usa ou quer adotar snapshot testing.

**Como instalar:**

```bash
mkdir -p {project_path}/.claude/skills/golden-tests
cp {framework_path}/skills/golden-tests/README.md {project_path}/.claude/skills/golden-tests/README.md
```

---
