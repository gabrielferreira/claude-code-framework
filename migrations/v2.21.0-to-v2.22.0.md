# Migration v2.21.0 → v2.22.0

> Data: 2026-04-09
> Tipo: minor

## Resumo

Adiciona hook de verificação pós-commit que roda `scripts/verify.sh` automaticamente após cada `git commit` via Claude Code. Zero tokens quando passa; injeta apenas linhas de erro (`❌`) quando falha. O `setup-framework` configura o hook automaticamente em `.claude/settings.local.json`; o `update-framework` avisa quando não está configurado.

## Pre-requisitos

- Versao atual do projeto: v2.21.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

### 🆕 Arquivos novos

#### 1. `docs/VERIFY_HOOK.md` (novo)

**O que e:** Documentação do hook pós-commit — explica por que usar, pré-requisitos, como o setup configura automaticamente, como configurar manualmente e como testar.

**Relevante se:** Projeto tem `scripts/verify.sh` e usa Claude Code para commits.

**Como instalar:** Copiar do framework para o projeto.

```bash
cp {framework_path}/docs/VERIFY_HOOK.md docs/VERIFY_HOOK.md
```

**Se decidir nao instalar:** Nenhuma consequência funcional — o hook funciona sem o doc. O doc serve de referência para configuração manual ou entendimento do mecanismo.

---

### ℹ️ Mudancas comportamentais (sem acao manual necessaria)

Estas mudancas afetam o comportamento do `/setup-framework` e `/update-framework`. Não requerem ação manual — entram em vigor na próxima vez que você rodar os skills.

**`/setup-framework` — novo passo 3.12:**
- Se `scripts/verify.sh` (ou `scripts/check.sh`) existir E `jq` estiver disponível: configura automaticamente o hook PostToolUse em `.claude/settings.local.json`
- Se `jq` não disponível: registra aviso no SETUP_REPORT.md com instrução para instalar
- Se nenhum script encontrado: silêncio

**`/update-framework` — novo check na Fase 5:**
- Se projeto tem verify.sh e hook ausente em settings.local.json: adiciona `💡` no UPDATE_REPORT.md apontando para `docs/VERIFY_HOOK.md`
- Se hook já configurado: silêncio
