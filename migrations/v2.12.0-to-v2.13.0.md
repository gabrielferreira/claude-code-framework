# Migration v2.12.0 → v2.13.0

> Data: 2026-04-06
> Tipo: minor

## Resumo

Analise de causa raiz profunda no PRD Creator (validacao do problema, porques encadeados, mapa causal, derivacao de comos, calibracao de escopo) + nova skill `/bug-report` para investigacao estruturada de bugs por times N2/N3 antes de escalar para engenharia.

## Pre-requisitos

- Versao atual do projeto: v2.12.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 📝 Structural (adicionar/remover secoes)

#### 1. `.claude/skills/prd-creator/SKILL.md`

**O que mudou:** Fluxo de analise de causa raiz reescrito de 6 para 9 passos. Novas etapas: Validacao do Problema (passo 2), Porques Encadeados com 3-5 niveis (passo 5), Mapa Causal (passo 6), Como Resolver Encadeado (passo 8), Calibracao de Escopo (passo 9). Tabela de verificacao pos-criacao expandida com novas secoes obrigatorias. Novas regras adicionadas.

**Recomendacao:** Substituir o conteudo do passo 5 (antigo) pelo novo fluxo de 9 passos. Se customizou perguntas especificas, integrar com os novos passos.

**Impacto se nao aplicar:** PRDs continuarao com analise rasa — problema aceito sem questionar, porques de um unico nivel, solucoes nao rastreadas para causas raiz.

#### 2. `.claude/prds/PRD_TEMPLATE.md`

**Secoes adicionadas:**

- `### Validacao do problema` (subsecao de `## Problema`)
- `## Mapa causal` (entre Porques e Quem e afetado)
- `## Calibracao de escopo` (entre Como resolver e Decisoes tomadas)

**Secoes alteradas:**

- `## Porques (analise de raiz)` → renomeado para `## Porques (analise de raiz encadeada)` — formato mudou de bullets planos para cadeias numeradas por causa com 3-5 niveis e resumo de causa raiz
- `## Como resolver` — cada acao agora inclui "Causa raiz atacada", "Cadeia de derivacao" (Como? → Como especificamente? → O que concretamente?), e "Sub-acoes"
- `## Verificacao pos-conclusao` — expandido de 4 para 10 items

**Impacto se nao aplicar:** Novos PRDs usarao o template antigo com estrutura rasa.

### 🆕 Arquivos novos

#### 6. `.claude/skills/bug-investigation/SKILL.md` (novo)

**O que e:** Skill `/bug-report` — guia investigacao estruturada de bugs em 10 passos (sintoma, validacao, comportamento esperado vs real, contexto da funcionalidade, reproducao, evidencias, porques encadeados, mapa de impacto, recomendacao para engenharia, calibracao de completude).

**Relevante se:** Projeto tem time de suporte N2/N3 que investiga bugs antes de escalar para engenharia.

**Como instalar:** Copiar do framework.

```bash
mkdir -p .claude/skills/bug-investigation
cp {framework}/skills/bug-investigation/SKILL.md .claude/skills/bug-investigation/SKILL.md
```

**Se decidir nao instalar:** Nenhum impacto — skill e opcional.

---

#### 7. `.claude/bugs/BUG_REPORT_TEMPLATE.md` (novo)

**O que e:** Template completo para relatorios de bug com todas as secoes de investigacao (sintoma, validacao, comportamento esperado vs real, contexto, reproducao, evidencias, porques encadeados, mapa de impacto, recomendacao, calibracao).

**Relevante se:** Vai usar a skill `/bug-report`.

**Como instalar:**

```bash
mkdir -p .claude/bugs
cp {framework}/bugs/BUG_REPORT_TEMPLATE.md .claude/bugs/BUG_REPORT_TEMPLATE.md
```

**Se decidir nao instalar:** Skill `/bug-report` nao tera template para copiar no modo repo.

---

#### 8. `docs/BUG_INVESTIGATION_PORTABLE_PROMPT.md` (novo)

**O que e:** Prompt standalone para investigacao de bugs em qualquer LLM (Gemini Gem, OpenAI GPT, Claude Project). Mesma metodologia da skill `/bug-report` sem dependencia do Claude Code.

**Relevante se:** Time de suporte usa LLMs fora do Claude Code para investigar bugs.

**Como instalar:**


**Se decidir nao instalar:** Nenhum impacto — doc e opcional.
