# Migration v2.23.1 → v2.24.0

> Data: 2026-04-09
> Tipo: minor

## Resumo

State machine com fases explícitas (research→plan→execute→verify→done) no STATE.md, gates de transição de status para specs, context-fresh execution com task-runner agent para despacho de tasks em sub-agents com contexto limpo, e task graph formalizado com grafo de dependências.

## Pre-requisitos

- Versao atual do projeto: v2.23.1
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 🆕 Arquivos novos

> Estes arquivos foram adicionados ao framework nesta versao.

#### 1. `.claude/agents/task-runner.md` (novo)

**O que e:** Agent que executa uma task individual de um spec com contexto limpo. Recebe briefing focado da sessão principal, implementa em worktree isolada, e reporta resultado (PASS/FAIL/PARTIAL).

**Relevante se:** o projeto usa sub-agents para implementação paralela (seção "Worktrees e subagents" no CLAUDE.md).

**Como instalar:** Copiar do framework source ou via `/update-framework`.

```bash
cp {framework}/agents/task-runner.md .claude/agents/task-runner.md
```

**Se decidir nao instalar:** sem impacto — o projeto continua sem context-fresh execution. A skill context-fresh referencia este agent.

#### 2. `.claude/skills/context-fresh/README.md` (novo)

**O que e:** Protocolo de orquestração para despachar tasks de um spec a sub-agents com contexto limpo (200k tokens cada). Define: como extrair tasks, compor briefings, despachar em waves, rastrear completion e integrar.

**Relevante se:** o projeto usa sub-agents e tem itens Médio+ com breakdown de tasks.

**Como instalar:**

```bash
mkdir -p .claude/skills/context-fresh
cp {framework}/skills/context-fresh/README.md .claude/skills/context-fresh/README.md
```

**Se decidir nao instalar:** sem impacto — o projeto continua delegando sub-agents sem protocolo formalizado.

### 📝 Structural — secoes novas/removidas

> Estes arquivos tem conteudo customizado pelo projeto. O `/update-framework` adiciona seções novas automaticamente.

#### 3. `.claude/specs/STATE.md` ⚠️ MANUAL (arquivo skip)

**ATENÇÃO:** STATE.md é `skip` no MANIFEST — o `/update-framework` NÃO toca neste arquivo. A seção precisa ser adicionada **manualmente**.

**Secao adicionada:** "Execução ativa" — adicionar entre "Blockers ativos" e "Lições aprendidas":

```markdown
## Execucao ativa

Item em andamento nesta sessao. Apenas 1 item ativo por vez (regra: um ciclo completo por vez).

| Campo | Valor |
|-------|-------|
| **Item** | {ID do backlog} — {titulo} |
| **Spec** | {path da spec ou URL Notion} |
| **Fase atual** | `research` · `plan` · `execute` · `verify` |
| **Entry criteria ✓** | {o que ja foi satisfeito para entrar nesta fase} |
| **Exit criteria pendente** | {o que falta para avancar para a proxima fase} |
| **Tasks** | {X/Y completas, ou N/A se Pequeno} |

> Fases por tamanho: **Pequeno** = `execute → verify → done`. **Medio** = `plan → execute → verify → done`. **Grande/Complexo** = `research → plan → execute → verify → done`.

### Log de transicoes

| De | Para | Quando | Motivo |
|----|------|--------|--------|
| — | {fase} | {data} | {inicio / criterio atingido / rollback} |
```

**Regras de manutenção a adicionar:**
- Ao iniciar item: preencher "Execução ativa" com fase inicial
- Ao avançar de fase: atualizar, registrar transição
- Ao concluir: limpar "Execução ativa"
- Ao retomar sessão: ler "Execução ativa" para saber onde parou

**Impacto se nao aplicar:** o projeto não terá tracking de fases — funciona, mas perde a capacidade de retomar trabalho interrompido.

#### 4. `.claude/skills/spec-driven/README.md`

**Secoes adicionadas:**

- "Fluxo de fases — state machine (Médio+)" — substitui "Fluxo RPI — Research, Plan, Implement (Médio+)". Tabela de fases com entry/exit criteria, mapeamento por tamanho, regras de transição.
- "Gates de transição de status" — validações obrigatórias antes de avançar status da spec (rascunho→aprovada→em andamento→concluída).

**Impacto se nao aplicar:** specs continuam mudando de status sem validação formal.

#### 7. `.claude/skills/execution-plan/README.md`
