# Migration v2.15.0 → v2.15.1

> Data: 2026-04-07
> Tipo: patch

## Resumo

Correcoes no setup para projetos Notion (nao criar backlog.md local, obrigar secao Integracao Notion no CLAUDE.md), renomeacao do slash command de `/spec-creator` para `/spec`, e avisos sobre nova sessao apos setup/update.

## Pre-requisitos

- Versao atual do projeto: v2.15.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 📝 Structural (adicionar/remover secoes)

#### 1. `.claude/skills/setup-framework/SKILL.md` (se instalado localmente)

**O que mudou:**
- Bloco 4 core agora inclui `spec-creator` e `backlog-update` explicitamente
- Nota diferenciando `spec-driven` de `spec-creator`
- Secao 3.5: caso Notion nao cria arquivos locais (backlog.md, TEMPLATE.md, etc.)
- Secao Notion no CLAUDE.md marcada como OBRIGATORIA
- Proximos passos: aviso sobre nova sessao

**Impacto se nao aplicar:** Setup pode pular skills essenciais e criar arquivos locais desnecessarios em projetos Notion.

#### 2. `.claude/skills/update-framework/SKILL.md` (se instalado localmente)

**O que mudou:**
- Categoria 3: nota diferenciando `spec-driven` de `spec-creator`
- Fase 3.4: aviso sobre nova sessao ao instalar slash commands novos
- Relatorio final: aviso sobre nova sessao

**Impacto se nao aplicar:** Update pode nao detectar falta de `spec-driven` e nao avisar sobre nova sessao.

### 🔄 Overwrite (substituir arquivo inteiro)

#### 3. `.claude/skills/spec-creator/SKILL.md`

**O que mudou:** Campo `name` no frontmatter renomeado de `spec-creator` para `spec`. O slash command agora e invocado como `/spec` (antes era `/spec-creator`).

**Como aplicar:** Abrir o arquivo e alterar linha 2:
```diff
- name: spec-creator
+ name: spec
```

**Impacto se nao aplicar:** Slash command continua sendo `/spec-creator` em vez de `/spec` (inconsistente com docs).

### 👀 Manual (decisao humana)

#### 4. Projetos com Notion: remover `backlog.md` local (se existir)

Se o projeto usa Notion para specs e o setup criou `.claude/specs/backlog.md` localmente, esse arquivo e desnecessario — o backlog vive no Notion.

**Acao sugerida:**
```bash
rm .claude/specs/backlog.md
# Opcional: remover tambem TEMPLATE.md, STATE.md, DESIGN_TEMPLATE.md se nao usa modo hibrido
```

#### 5. Projetos com Notion: verificar secao `## Integracao Notion (specs)` no CLAUDE.md

Se o CLAUDE.md do projeto NAO tem a secao `## Integracao Notion (specs)`, a `/spec` opera em modo local. Para ativar modo Notion, adicionar a secao com a configuracao da database.
