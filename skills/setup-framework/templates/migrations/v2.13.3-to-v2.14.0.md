# Migration v2.13.3 → v2.14.0

> Data: 2026-04-07
> Tipo: minor

## Resumo

Setup agora analisa o codigo-fonte real do projeto para detectar padroes (logging, errors, HTTP, etc.) e customiza skills com exemplos reais. Ambos setup e update ganham auditoria de relevancia (Categoria 6) que valida se o conteudo instalado faz sentido pro projeto. PRD Creator ganha diagramas e Bug Investigation ganha criacao de cards no Jira.

## Pre-requisitos

- Versao atual do projeto: v2.13.3
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 📝 Structural (adicionar/remover secoes)

#### 1. `.claude/skills/update-framework/SKILL.md`

**O que mudou:**
- Nova Fase 0.6 "Detectar padroes de codigo (CODE_PATTERNS)" — analisa codigo-fonte real para detectar libs de logging, error handling, HTTP client, validacao, ORM e config
- Nova Categoria 6 "Relevancia de conteudo" na auditoria 5b com subsecoes:
  - 6.1: Exemplos de codigo incompativeis com a stack
  - 6.2: Libs e padroes divergentes dos detectados
  - 6.3: Skills e agents irrelevantes para o tipo de projeto
  - 6.4: Secoes do CLAUDE.md irrelevantes
  - 6.5: Docs irrelevantes
  - 6.6: Evolucao do projeto (detecta mudancas desde ultimo setup)
  - 6.7: Procedimento de remocao completa

**Como aplicar:** O `/update-framework` aplicara como structural — adicionando as novas secoes sem alterar customizacoes existentes.

**Impacto se nao aplicar:** O update nao validara se skills/agents/docs fazem sentido para o projeto. Conteudo generico ou irrelevante permanecera sem aviso.

---

#### 2. `.claude/skills/prd-creator/SKILL.md`

**O que mudou:**
- Novo passo de geracao de diagrama causal (.mmd) com output Mermaid
- Export para ferramentas visuais (Miro, FigJam, Excalidraw)
- Geracao de arquivo .mmd standalone por padrao

**Como aplicar:** O `/update-framework` aplicara como structural.

---

#### 3. `.claude/skills/bug-investigation/SKILL.md`

**O que mudou:**
- Novo passo de criacao automatica de card no Jira apos investigacao concluida

**Como aplicar:** O `/update-framework` aplicara como structural.

---

### 👀 Revisao manual necessaria

#### 4. `CLAUDE.md`

**O que mudou no template:** Regra 11 atualizada para refletir 6 categorias de auditoria (antes eram 5), incluindo a nova Categoria 6 de relevancia de conteudo.

**Acao:** Se voce customizou regras de auditoria ou tem instrucoes sobre validacao de conteudo no CLAUDE.md, revisar se a nova regra se aplica.

---

## Notas

- A Categoria 6 de relevancia roda automaticamente em toda execucao do `/update-framework`, mesmo sem mudanca de versao. Isso captura problemas que passaram no setup original.
- A deteccao de CODE_PATTERNS e best-effort — se o projeto tem poucos arquivos, a etapa e pulada automaticamente.
- Nenhuma correcao da Categoria 6 e automatica — sempre pergunta ao usuario com opcoes claras.
