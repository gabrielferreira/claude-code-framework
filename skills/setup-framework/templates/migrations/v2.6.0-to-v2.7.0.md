# Migration v2.6.0 → v2.7.0

> Data: 2026-04-02
> Tipo: minor

## Resumo

Release focada em PRDs: diretorio proprio para PRDs (`prds/`), doc portavel standalone, flags `--from` e `--export` na skill `/prd`, verificacao pos-criacao obrigatoria no `/spec` e `/prd`, e secao de regras imperativas no CLAUDE.md. Projetos que instalaram PRDs na v2.6.0 precisam migrar os arquivos de `.claude/specs/` para `.claude/prds/`.

## Pre-requisitos

- Versao atual do projeto: v2.6.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## ⚠️ Migracao de PRDs (se aplicavel)

> Esta secao e relevante apenas para projetos que instalaram PRDs na v2.6.0 e tem arquivos em `.claude/specs/prd-*.md` ou `.claude/specs/PRD_TEMPLATE.md`.

### O que mudou

A partir da v2.7.0, PRDs vivem em `.claude/prds/` (diretorio proprio) em vez de `.claude/specs/`. Isso separa os dois tipos de artefato e permite indices e templates independentes.

### Como migrar

**Passo 1: Criar o diretorio `prds/`**

```bash
mkdir -p {project_path}/.claude/prds
```

**Passo 2: Mover o template de PRD**

```bash
# Mover o template para o novo local
mv {project_path}/.claude/specs/PRD_TEMPLATE.md {project_path}/.claude/prds/PRD_TEMPLATE.md
```

**Passo 3: Mover os PRDs existentes**

```bash
# Listar PRDs no local antigo
ls {project_path}/.claude/specs/prd-*.md 2>/dev/null

# Mover cada PRD para o novo diretorio
mv {project_path}/.claude/specs/prd-*.md {project_path}/.claude/prds/
```

**Passo 4: Criar o indice de PRDs**

```bash
cp {framework_path}/PRDS_INDEX.template.md {project_path}/.claude/prds/PRDS_INDEX.md
```

Editar `.claude/prds/PRDS_INDEX.md` para listar os PRDs migrados.

**Passo 5: Atualizar referencias**

Se o `CLAUDE.md` do projeto menciona o caminho `.claude/specs/PRD_TEMPLATE.md`, atualizar para `.claude/prds/PRD_TEMPLATE.md`. Fazer o mesmo em qualquer outro documento que referencie o caminho antigo.

```bash
# Verificar referencias ao caminho antigo
grep -r "specs/PRD_TEMPLATE" {project_path} --include="*.md"
grep -r "specs/prd-" {project_path} --include="*.md"
```

**Passo 6: Verificar**

```bash
# Confirmar que nao ha PRDs no local antigo
ls {project_path}/.claude/specs/prd-*.md 2>/dev/null || echo "OK — nenhum PRD no local antigo"

# Confirmar que os PRDs estao no novo local
ls {project_path}/.claude/prds/
```

### Se decidir nao migrar agora

A skill `/prd` na v2.7.0 passa a gravar em `.claude/prds/` por padrao. Se nao migrar, os PRDs antigos continuarao existindo em `.claude/specs/` (nao sao deletados automaticamente), mas novos PRDs criados pela skill irao para `.claude/prds/`. Isso resulta em PRDs espalhados nos dois diretorios — recomendamos migrar para manter consistencia.
