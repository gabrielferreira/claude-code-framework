# Migration v2.8.0 → v2.9.0

> Data: 2026-04-02
> Tipo: minor

## Resumo

Adiciona os campos **Autor**, **Responsavel** e **Concluida em** ao template de spec e ao ciclo de vida. A skill `/spec` agora preenche Autor na criacao, e a skill `/definition-of-done` preenche Responsavel e Concluida em ao marcar uma spec como concluida. A resolucao de identidade usa `notion-get-users self` no modo Notion e `git config user.name` no modo repo. Nova regra: o campo "Arquivo" no Notion deve ficar vazio quando a spec vive no proprio Notion.

## Pre-requisitos

- Versao atual do projeto: v2.8.0
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

### 📝 Structural (adicionar/remover secoes)

> Estes arquivos tem conteudo customizado pelo projeto. Apenas secoes novas/removidas sao listadas.

#### 1. `.claude/skills/spec-creator/README.md`

**Secoes adicionadas:**

Adicionar na etapa de criacao de spec (apos resolver titulo/objetivo, antes de gravar o arquivo):

```markdown
## Resolucao de identidade (Autor)

Ao criar uma spec, resolver o autor da seguinte forma:

- **Modo Notion:** chamar `notion-get-users self` e usar o `name` retornado.
- **Modo repo:** executar `git config user.name` e usar o valor retornado.

Preencher o campo `Autor` com o valor resolvido. Se nao for possivel resolver (sem MCP configurado e sem git), deixar o campo vazio.
```

Adicionar na regra sobre o campo "Arquivo" (modo Notion):

```markdown
## Campo "Arquivo" no Notion

Quando a spec vive inteiramente no Notion (conteudo editado diretamente na pagina), deixar o campo "Arquivo" vazio na database. Preencher apenas quando a spec tem um arquivo `.md` correspondente no repositorio.
```

**Impacto se nao aplicar:** O campo Autor nao e preenchido automaticamente na criacao — dados de autoria ficam em branco ate preenchimento manual.

#### 2. `.claude/skills/definition-of-done/README.md`

**Secoes adicionadas:**

Adicionar na etapa de conclusao de spec (ao marcar como concluida):

```markdown
## Preenchimento de campos de conclusao

Ao marcar uma spec como concluida (Definition of Done aprovado), preencher:

- **Responsavel:** resolver identidade igual ao campo Autor (modo Notion: `notion-get-users self`; modo repo: `git config user.name`).
- **Concluida em:** data atual no formato `YYYY-MM-DD`.

Estes campos devem ser preenchidos antes de registrar o status final da spec.
```

Adicionar na verificacao pos-implementacao:

```markdown
## Checklist de campos de conclusao

Apos marcar spec como concluida, verificar:

- [ ] Campo `Responsavel` preenchido com nome do responsavel pela conclusao
- [ ] Campo `Concluida em` preenchido com a data de conclusao
- [ ] Status da spec atualizado para "Concluida" (ou equivalente configurado no projeto)
```

**Impacto se nao aplicar:** Specs marcadas como concluidas ficam sem data de conclusao e sem responsavel — rastreabilidade e metricas de ciclo de vida comprometidas.
