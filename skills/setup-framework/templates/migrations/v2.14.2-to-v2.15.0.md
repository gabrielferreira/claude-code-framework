# Migration v2.14.2 → v2.15.0

> Data: 2026-04-07
> Tipo: minor

## Resumo

Suporte completo a monorepo no setup e update: distribuicao de skills/agents por sub-projeto (L0 vs L2), CODE_PATTERNS por sub-projeto (mesmo linguagem com padroes diferentes), isolamento de verify.sh/docs/CLAUDE.md L2, e modelo misto onde sub-projetos herdam da raiz e so listam overrides.

## Pre-requisitos

- Versao atual do projeto: v2.14.2
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 📝 Structural (adicionar/remover secoes)

> Estes arquivos tem conteudo customizado pelo projeto. Apenas secoes novas/removidas sao listadas.

#### 1. `.claude/skills/setup-framework/SKILL.md`

**Secoes adicionadas:**

As seguintes secoes foram adicionadas para suporte a monorepo:

- **Fase 2.5 — Monorepo: distribuicao de skills/agents**: Detecta sub-projetos, classifica skills como L0 (universal/processo) ou L2 (stack-specific), oferece 3 opcoes de organizacao (por sub-projeto, seccionado, por skill).
- **Fase 2.5 — CODE_PATTERNS por sub-projeto**: Roda CODE_PATTERNS independente em cada sub-projeto, mesmo quando compartilham linguagem (ex: dois projetos Go com libs diferentes).
- **Fase 3 — Monorepo: geracao de CLAUDE.md L2**: Template de CLAUDE.md L2 com modelo misto — sub-projetos so listam skills que diferem da raiz, o resto herda via concatenacao L0+L2.
- **Fase 3 — Monorepo: verify.sh e docs por sub-projeto**: Cada sub-projeto gera seu proprio verify.sh, com orquestrador na raiz opcional.
- **Fase 3 — Monorepo: identificacao de arquivos com mesmo nome**: Skills/agents/docs com mesmo nome em sub-projetos diferentes sao identificados pelo path completo.

**Impacto se nao aplicar:** Setup continuara funcionando para single-repo, mas monorepos terao skills genericas sem customizacao por sub-projeto.

#### 2. `.claude/skills/update-framework/SKILL.md`

**Secoes adicionadas:**

- **Fase 4 — Monorepo: atualizacao por sub-projeto**: Aplica diff em cada sub-projeto usando seus proprios CODE_PATTERNS. Detecta mudancas de stack (ex: migrou de zap para slog) e sugere migracao de padroes.
- **Fase 4 — Monorepo: verify.sh por sub-projeto**: Atualiza verify.sh de cada sub-projeto independentemente.
- **Fase 4 — Monorepo: identificacao de mesmo nome**: Usa path completo para diferenciar arquivos com mesmo nome entre sub-projetos.

**Impacto se nao aplicar:** Update continuara funcionando para single-repo, mas monorepos nao terao atualizacao inteligente por sub-projeto.
