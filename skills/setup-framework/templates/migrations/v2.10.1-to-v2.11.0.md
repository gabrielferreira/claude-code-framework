# Migration v2.10.1 → v2.11.0

> Data: 2026-04-06
> Tipo: minor

## Resumo

Adiciona sistema de migrations para atualizacoes manuais entre versoes (como migrations de banco de dados) e backlog do framework com analise do ecossistema SDD.

## Pre-requisitos

- Versao atual do projeto: v2.10.1
- Se estiver em versao anterior, aplique as migrations anteriores primeiro

## Mudancas

### 🆕 Arquivos novos

#### 1. `migrations/README.md` (novo)

**O que e:** Explicacao do sistema de migrations — quando usar, como usar, convencao de nomes.

**Relevante se:** Sempre. E a documentacao do sistema de migrations.

**Como instalar:**
```bash
mkdir -p migrations
cp {framework}/migrations/README.md migrations/README.md
```

**Se decidir nao instalar:** Sem impacto funcional — e apenas documentacao.

#### 2. `migrations/MIGRATION_TEMPLATE.md` (novo)

**O que e:** Template padrao usado para gerar novas migrations durante o release.

**Relevante se:** Sempre. Serve de referencia para entender o formato.

**Como instalar:**
```bash
cp {framework}/migrations/MIGRATION_TEMPLATE.md migrations/MIGRATION_TEMPLATE.md
```

**Se decidir nao instalar:** Sem impacto — usado apenas como referencia.

#### 3. `migrations/v2.4.0-to-v2.5.0.md` ate `migrations/v2.10.0-to-v2.10.1.md` (novos)

**O que e:** 7 migrations retroativas cobrindo todas as versoes de v2.4.0 ate v2.10.1. Cada arquivo descreve o que mudou e como aplicar manualmente.

**Relevante se:** Sempre — servem como historico de mudancas e referencia.

**Como instalar:**
```bash
cp {framework}/migrations/v2.*.md migrations/
```

**Se decidir nao instalar:** Perde o historico de migrations. Pode copiar depois se precisar.
