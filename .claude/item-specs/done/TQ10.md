# TQ10 — Item-specs retroativas para DF13 e TQ5

**Contexto:** auditoria rodada em 2026-04-16 identificou que `.claude/item-specs/INDEX.md` listava DF13 (Discovery Routing + `/quick`, v2.36.0) e TQ5 (Seções obrigatórias nas skills, v2.31.0) como "Concluídos" mas com `—` no lugar do link. Os arquivos `done/DF13.md` e `done/TQ5.md` nunca foram criados. Outros itens das mesmas releases (SA4, SW7, AU4) tinham spec — era gap, não padrão intencional. Ambos foram mudanças de design relevantes; sem spec retroativa, sessão futura que quisesse entender "por que `/quick` foi desenhada assim?" dependeria só de git log + CHANGELOG.

**Abordagem:** reconstituir as duas specs a partir do CHANGELOG, commits (`f22694f feat: slash command /quick (DF13)`, `f62699d fix: TQ5 — seções obrigatórias`) e do estado atual dos artefatos (`skills/quick/SKILL.md`, `scripts/validate-structure.sh`). Seguir o mesmo formato dos outros arquivos em `done/` (Contexto / Abordagem / Critérios / Restrições). Atualizar `INDEX.md` substituindo `—` pelos links.

Alternativas descartadas:
- Marcar `—` como "sem spec (implementado diretamente)" no INDEX — seria sinalizar gap como intencional sem resolver a rastreabilidade.
- Descartar o item — valor baixo se cada item tivesse versão em log, mas sessões futuras perdem acesso rápido às decisões.

**Critérios de aceitação:**
- [x] `.claude/item-specs/done/DF13.md` criado com formato padrão
- [x] `.claude/item-specs/done/TQ5.md` criado com formato padrão
- [x] `.claude/item-specs/INDEX.md` atualizado com links para as duas specs
- [x] Cada spec cita ao menos: motivação, decisão, alternativas descartadas, critérios que foram cumpridos, restrições
- [x] Specs baseadas em evidência verificável (CHANGELOG, git log, estado dos artefatos) — não especulação

**Restrições:**
- Specs retroativas declaram o estado real do item, não reinventam decisões. Se a decisão original não é recuperável de forma confiável, sinalizar "reconstituído a partir do artefato final" em vez de inventar.
- Não aplicar retroação massiva para outros itens — só reagir a gaps específicos identificados em auditoria.
- `INDEX.md` deve manter `—` apenas para itens intencionalmente sem spec (política a estabelecer se o caso voltar).
